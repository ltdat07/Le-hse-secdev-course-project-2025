# Security policy

**Важно:** не храните секреты/ключи/пароли в репозитории. Если случайно закоммитили секрет — инвалидация ключа у провайдера обязательна.

## Контакт для репорта
- Primary contact: `@ltdat07` (GitHub)
- Дополнительно (если удобно): `danielle172004@gmail.com`

## Как сообщать об уязвимости
Пожалуйста, отправляйте репорты приватно (не создавайте публичный issue).

## Что делать, если обнаружены секреты в репозитории
1. Немедленно инвалидация ключа/секрета у провайдера (rotate/revoke).
2. Свяжитесь с контактами выше и опишите обнаружение (commit, файл).
3. Если нужны инструкции по очистке истории — напишите нам; избегайте публикации PoC публично.

## Политика данных
- Во время курса используйте только синтетические/тестовые данные (не храните ПДн/платежные данные).
- В коде используйте единый формат ошибок (см. README).
